Getting Started

:date_long
AuthProbe helps you isolate exactly where MCP OAuth fails. Use this path to onboard quickly: Install the AuthProbe CLI. Run the quickstart scan command. Use verification commands to confirm your fix. Continue with Install and Quickstart.

Install

:date_long
Install AuthProbe using your preferred package workflow for your environment. After installation, verify the CLI is available: authprobe --help Then continue to Quickstart.

Quickstart

:date_long
Run your first scan against an MCP endpoint with OAuth enabled: authprobe scan --target https://mcp.example.com --auth oauth --verbose Review the findings to identify where the OAuth exchange fails, then run the provided verification commands after remediation.

CI

:date_long
Run AuthProbe in CI to detect MCP OAuth regressions before deployment. Use deterministic findings to fail builds only on real protocol breakpoints.

Troubleshooting

:date_long
If scans fail unexpectedly, start by rerunning with verbose output and checking the deterministic findings. This page is a starter stub and can be expanded with common failure signatures and fixes.
